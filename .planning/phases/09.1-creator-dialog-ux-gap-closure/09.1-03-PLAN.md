<objective>
F5 — Auto-save creator dialog state to localStorage and restore on reopen, so that closing the dialog or refreshing the page does not lose in-progress work.

Implements 09.1-CONTEXT.md decisions D-15..D-22:
- D-15/D-18: localStorage key `"xareable.postCreator.draft"`, JSON object with all D-16 fields plus `savedAt` ISO timestamp
- D-16: persist `contentType`, `step`, `referenceText`, `slideCount`, `postMood`, `aspectRatio`, `imageResolution`, `videoDuration`, `videoResolution`, `useText`, `copyText`, `selectedTextStyleIds`, `useLogo`, `logoPosition`, `contentLanguage`, `sceneryId`
- D-17: NEVER persist `referenceImages` or `enhancementFile` (binary File objects)
- D-19: 7-day TTL — drafts older than 7 days are deleted silently on open
- D-20: debounced 500ms save on every state change; on open, if a fresh draft exists show a banner with Continue / Start fresh; if no draft or expired, no banner
- D-21: cleanup on success (any generation completes — image, video, carousel, enhancement)
- D-22: cleanup on Save & Close / Generate Another / dialog close after a successful run

Purpose: User reported (Phase 9 HUMAN-UAT Test 7 follow-up) that accidentally closing or refreshing wipes their in-progress carousel/enhancement setup. This is a quality-of-life fix that makes the creator dialog feel persistent without committing to a backend draft system.

Output: Stateful dialog that survives close/reopen across the same browser session and across page reloads (within 7 days). No backend changes.

This plan is **wave 2** because it modifies the same `post-creator-dialog.tsx` file that 09.1-02 modifies (F1+F2+F3 visual fixes). Sequencing avoids merge conflict on shared file ownership.
</objective>

    ```typescript
    // ── F5: draft persistence (09.1 D-15..D-22) ─────────────────────────────────
    const DRAFT_STORAGE_KEY = "xareable.postCreator.draft";
    const DRAFT_TTL_MS = 7 * 24 * 60 * 60 * 1000; // D-19: 7 days
    const DRAFT_DEBOUNCE_MS = 500; // D-20

    type CreatorDraft = {
      savedAt: string;
      contentType: string;
      step: number;
      referenceText: string;
      slideCount: number;
      postMood: string;
      aspectRatio: string;
      imageResolution: "512px" | "1K" | "2K" | "4K";
      videoDuration: "4" | "6" | "8";
      videoResolution: "720p" | "1080p" | "4k";
      useText: boolean;
      copyText: string;
      selectedTextStyleIds: string[];
      useLogo: boolean;
      logoPosition: string;
      contentLanguage: string;
      sceneryId: string | null;
    };

    function loadDraft(): CreatorDraft | null {
      try {
        const raw = localStorage.getItem(DRAFT_STORAGE_KEY);
        if (!raw) return null;
        const parsed = JSON.parse(raw) as CreatorDraft;
        if (typeof parsed?.savedAt !== "string") {
          localStorage.removeItem(DRAFT_STORAGE_KEY);
          return null;
        }
        const ageMs = Date.now() - new Date(parsed.savedAt).getTime();
        if (Number.isNaN(ageMs) || ageMs > DRAFT_TTL_MS) {
          // D-19: silent expiry
          localStorage.removeItem(DRAFT_STORAGE_KEY);
          return null;
        }
        return parsed;
      } catch {
        // Corrupt JSON — wipe and start fresh
        try { localStorage.removeItem(DRAFT_STORAGE_KEY); } catch {}
        return null;
      }
    }

    function clearDraft(): void {
      try { localStorage.removeItem(DRAFT_STORAGE_KEY); } catch {}
    }

    function saveDraft(draft: Omit<CreatorDraft, "savedAt">): void {
      try {
        const payload: CreatorDraft = { ...draft, savedAt: new Date().toISOString() };
        localStorage.setItem(DRAFT_STORAGE_KEY, JSON.stringify(payload));
      } catch {
        // QuotaExceededError or storage disabled — silently no-op
      }
    }
    ```
