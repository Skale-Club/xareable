<decisions>
## Implementation Decisions

### F1: Carousel result thumbnail sizing (responsive)

- **D-01:** The result view's slide grid (Component Inventory §6 in 09-UI-SPEC) uses fixed `w-[72px] h-[72px]` placeholders today. Replace with a responsive grid where slide thumbnails grow when there are fewer slides and shrink when there are more, fitting comfortably in the dialog body.
- **D-02:** Recommended sizing rule — column count scales inversely with slide count to keep thumbnails as large as possible:
  - 3 slides → `grid-cols-3` (large thumbnails ~ aspect-square fitting dialog width)
  - 4 slides → `grid-cols-4`
  - 5–6 slides → `grid-cols-3` (2-3 row split, slightly smaller)
  - 7–8 slides → `grid-cols-4` (2-row split, smaller)
  Use `aspect-square` for each tile so they always render proportionally; let the grid resolve actual pixel size.
- **D-03:** Tile internal layout: thumbnail image fills the tile, overlay label `"Slide N"` at bottom-left for accessibility (matches existing pattern).

### F2: Carousel slide hover preview

- **D-04:** Hovering any slide thumbnail in the result view shows a larger floating preview anchored near the cursor (or a fixed corner — Claude's Discretion if cursor-tracking is fragile). Preview is the same image enlarged, ~50-60% of dialog width.
- **D-05 (Claude's Discretion):** Implementation simpler-first — anchor the floating preview to a fixed position (e.g. center-right of the dialog) on hover, with a small fade-in animation. If cursor-tracking is needed, use absolute positioning with `mousemove` event throttled to ~60fps. Start with the simpler fixed-anchor approach; iterate to cursor-following only if user requests after seeing it live.
- **D-06:** Preview uses `pointer-events-none` so it doesn't block other interactions; appears with `framer-motion` fade for consistency with the rest of the dialog.

### F3: Scenery picker compact grid

- **D-07:** Replace the current `grid grid-cols-1 sm:grid-cols-2 xl:grid-cols-3 gap-4` in the Scenery Picker step (UI-SPEC Component Inventory §3) with a denser layout: `grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 gap-3`.
- **D-08:** Reduce card padding (`p-2` instead of `p-3`) and use a smaller thumbnail (`aspect-video` stays but the card overall is smaller). Keep the same selection styling (`border-violet-400 bg-violet-400/10`).
- **D-09:** Truncate `prompt_snippet` to a single line (`line-clamp-1` instead of `line-clamp-2`) to keep cards short.

### F4: Enhancement caption generation (re-spec ENHC-08)

- **D-10:** Re-spec ENHC-08. The original spec said "Enhancement posts never run logo overlay or caption quality post-processing". Updated: **Enhancement posts DO generate a caption** (no logo overlay, no on-image text — those still skip). Caption generation runs as a final step after the image-model edit call and before the storage upload.
- **D-11:** Backend implementation in `server/services/enhancement.service.ts`:
  - After `callEnhancementImageModel` returns the result image
  - Add a new `generateEnhancementCaption(scenery, contentLanguage, apiKey)` function that calls the existing text model (`gemini-2.5-flash`) with a prompt asking for a 1-2 sentence Instagram caption tailored to the product and the scenery context (English-default per CRTR-06 / project constraint, language-aware via `contentLanguage` if frontend ever passes it — currently hardcoded to "en" per Phase 7 D-XX, so MVP just uses English)
  - Caption is saved on the `posts` row (`caption` column) instead of `null`
  - Token totals from the caption call are summed into `tokenTotals.textInputTokens / textOutputTokens` before the result is returned
- **D-12:** SSE complete event includes the caption: `{ type: "complete", post, image_url, caption }`. The route at `server/routes/enhance.routes.ts` already passes through whatever the service returns; just ensure the service emits the caption in the complete event.
- **D-13:** Frontend (`post-creator-dialog.tsx`): the existing `handleGenerateEnhancement` `onComplete` already destructures `caption` from the SSE payload and passes it to `openViewer({ ..., caption })` — minimal change. Verify the path works end-to-end after the backend lands.
- **D-14:** Update REQUIREMENTS.md ENHC-08 to reflect the new spec: "Enhancement posts never run logo overlay or on-image text composition — but DO generate a plain Instagram-ready caption."

### F5: Creator draft auto-save

- **D-15:** Persist creator dialog state to `localStorage` so that closing/reopening (or accidental refresh) restores the user's in-progress work.
- **D-16 (scope of persistence):** Persist textual / selection state only:
  - `contentType`, `step`, `referenceText`, `slideCount`, `postMood`, `aspectRatio`, `imageResolution`, `videoDuration`, `videoResolution`, `useText`, `copyText`, `selectedTextStyleIds`, `useLogo`, `logoPosition`, `contentLanguage`, `sceneryId`
- **D-17 (NOT persisted):**
  - `referenceImages` (binary File objects) — too large + impractical to serialize
  - `enhancementFile` (binary File object) — same reason
  Note: when restoring, file fields are empty; user has to re-upload. Acceptable trade-off.
- **D-18 (storage key + structure):** `localStorage` key: `"xareable.postCreator.draft"`. Stored value: JSON object with all D-16 fields plus a `savedAt: ISO string` timestamp.
- **D-19 (TTL):** Drafts expire after 7 days. On dialog open, check `savedAt` — if > 7 days old, delete the draft and start fresh without prompting.
- **D-20 (UX flow):**
  - Save: every state change debounced (500ms) writes to localStorage
  - Restore on dialog open:
    - If a draft exists AND it's < 7 days old → show inline banner above the steps: "Continue where you left off?" with two buttons: `"Continue draft"` (loads the state) and `"Start fresh"` (deletes draft and starts clean)
    - If no draft or expired → no banner, normal fresh start
- **D-21 (cleanup on success):** When generation completes successfully (carousel, enhancement, or image/video), delete the draft from localStorage.
- **D-22 (cleanup on user action):** "Save & Close" / "Generate Another" / closing the dialog after success → delete the draft.

### Claude's Discretion

- **D-05** (hover preview implementation: fixed-anchor vs cursor-tracking) — start simple, iterate
- **D-19 TTL** — 7 days is reasonable; user didn't specify

### Folded Todos

None — all issues are user-reported from HUMAN-UAT.
