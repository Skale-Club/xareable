### 7. Cache Invalidation on SSE Events (GLRY-05)

**Current pattern:** `posts.tsx` re-fetches by watching `createdVersion` in the `useEffect` dependency array (line 232). `markCreated()` is called from `post-creator-dialog.tsx` on SSE `complete`.

**Required change:** `markCreated()` must also be called on SSE `error` events from the carousel route so partial-draft carousels appear in the gallery immediately.

**Implementation site:** `post-creator-dialog.tsx` carousel generate handler ŌĆö in the `fetchSSE` `onEvent` callback, add `markCreated()` call when event type is `"error"` (in addition to the existing `"complete"` call). This is a one-line change at the SSE handler level, not a TanStack Query invalidation ŌĆö the existing `createdVersion` increment mechanism is sufficient.

**No `useQueryClient` / `invalidateQueries` calls are introduced.** The project uses a manual `refreshTick` / `createdVersion` pattern, not TanStack Query cache invalidation. Stay consistent with the established pattern.

### TypeScript Exhaustiveness Guard (GLRY-04)

Location: inside the `getContentTypeIcon(contentType)` helper function in `posts.tsx` (or an inline switch in the JSX, converted to a named function).

Pattern:

```ts
function assertNever(x: never): never {
  throw new Error(`Unhandled content_type: ${x}`);
}

function getContentTypeIcon(contentType: Post["content_type"]) {
  switch (contentType) {
    case "image": return <ImageIcon ... />;
    case "video": return <VideoIcon ... />;
    case "carousel": return <LayoutPanelTop ... />;
    case "enhancement": return <Sparkles ... />;
    default: return assertNever(contentType);
  }
}
```

TypeScript will produce a compile error in the `default` branch whenever a new `content_type` enum value is added to `shared/schema.ts` without updating this switch. This satisfies GLRY-04.

The same `assertNever` guard pattern is applied in `PostViewerDialog` wherever `content_type` determines rendering behavior.
