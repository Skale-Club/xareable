<objective>
Implement the server-side carousel slide edit endpoint. Adds `POST /api/carousel/slide/edit` to `server/routes/carousel.routes.ts`. The endpoint mirrors `edit.routes.ts` but: (a) targets a single slide by `slide_id`, (b) persists into `post_slide_versions`, (c) passes slide-1 as `additionalRefs[0]` for slides 2..N to anchor visual style, (d) bills 1× edit cost regardless of slide count, (e) handles `source: "quick_remake"` by injecting `post.ai_prompt_used` as the regeneration seed.

Purpose: Provide the missing server endpoint that the carousel-slide PostEditDialog variant (Plan 13-03) and the viewer Quick Remake handler (Plan 13-04) target.
Output: One new route handler appended to carousel.routes.ts plus active CRSL-EDIT-03/04/05 verify checks.
</objective>

Append a NEW route handler `router.post("/api/carousel/slide/edit", ...)` to `server/routes/carousel.routes.ts`. Do NOT modify `server/routes/edit.routes.ts` (RESEARCH.md Pitfall 6 + Open Question 4).

Structure — mirror edit.routes.ts (image branch only — slides are always images, never videos):

1. **Auth + parsing**:
   - Extract Bearer token, call `createServerSupabase(token).auth.getUser(token)`. Return 401 on failure.
   - Validate body with `editSlideRequestSchema.safeParse` (from Plan 13-01). Return 400 on failure.
   - Destructure `{ slide_id, post_id, edit_prompt, content_language, source, edit_context }`.

2. **Ownership + slide fetch**:
   - SELECT slide row from `post_slides` joined to `posts` ownership check:
     ```typescript
     const { data: slide } = await supabase
       .from("post_slides")
       .select("id, post_id, slide_number, image_url, posts!inner(id, user_id, content_type, ai_prompt_used)")
       .eq("id", slide_id)
       .eq("posts.user_id", user.id)
       .single();
     ```
   - If `!slide` → 404. If `slide.post_id !== post_id` → 403 (mismatch). If `slide.posts.content_type !== "carousel"` → 400 ("Not a carousel slide").

3. **Profile + key resolution + brand fetch** — copy-paste from edit.routes.ts lines 136–162 (select `is_admin, is_affiliate, api_key, openai_api_key, image_provider`; get geminiApiKey via `getGeminiApiKey`; fetch brand for company_name/colors/mood/logo_url).

4. **Credits gate** — CRITICAL (RESEARCH.md Pitfall 2):
   - Use `checkCredits(user.id, "edit")` with NO slideCount argument. Identical to edit.routes.ts line 165.
   - Apply same denial-reason → 402 mapping (insufficient_credits / usage_budget_reached / upgrade_required / subscription_required).
   - If `source === "quick_remake"` and `!ownApiKey`, also gate via `canUseQuickRemake(user.id)`; return 402 `quick_remake_limit_reached` on denial.

5. **effectiveEditContext for quick_remake** — copy exactly from edit.routes.ts lines 94–116 (defaults goal_text/focus_areas/focus_details when source=="quick_remake" and edit_context not provided).

6. **SSE init**:
   - `const sse = initSSE(res); sse.startHeartbeat();`
   - `sse.sendProgress("auth", "Verified. Starting slide edit...", 10);`
   - 280s safety timer mirroring edit.routes.ts lines 244-252; log to `generation_logs` on timeout via a local `logSlideEditError` helper modeled on `logEditError`.

7. **Fetch current slide image + (if slide_number > 1) slide-1 image as anchor**:
   ```typescript
   sse.sendProgress("image_generation", "Loading slide images...", 20);

   const currentResp = await fetch(slide.image_url);
   const currentBuf = Buffer.from(await currentResp.arrayBuffer());
   const currentBase64 = currentBuf.toString("base64");
   const currentMime = currentResp.headers.get("content-type")?.split(";")[0] || "image/png";

   let slide1Ref: { mimeType: string; data: string } | undefined;
   if (slide.slide_number > 1) {
     const { data: s1 } = await supabase
       .from("post_slides")
       .select("image_url")
       .eq("post_id", post_id)
       .eq("slide_number", 1)
       .single();
     if (s1?.image_url) {
       const r = await fetch(s1.image_url);
       const buf = Buffer.from(await r.arrayBuffer());
       slide1Ref = {
         mimeType: r.headers.get("content-type")?.split(";")[0] || "image/png",
         data: buf.toString("base64"),
       };
     }
   }
   ```

8. **Build edit prompt** — copy structured-instructions builder from edit.routes.ts lines 348–411 (textEditRules, structuredEditInstructions, language instruction, brand-context preamble). Append a carousel-specific suffix:
   ```
   Carousel context: this is slide ${slide.slide_number} of a multi-slide carousel for "${brand.company_name}". Preserve the carousel's overall visual language. ${slide1Ref ? "A reference image showing slide 1 is provided — match its visual style, color palette, and typographic tone." : "This IS slide 1 — your output sets the visual language for the rest of the carousel."}
   ```
   For Phase 13 v1, the Text-on-Image step is omitted (CRSL-10), so `text_mode` will typically be `"keep"`; do NOT run `enforceExactImageText` (which is for text-rendering posts only).

9. **Provider edit call**:
   ```typescript
   sse.sendProgress("image_generation", "Editing slide...", 35);
   const provider = await getActiveImageProvider(editProfile);
   let imageApiKey = geminiApiKey;
   if (provider.name === "openai") {
     const r = await getOpenAIApiKey(editProfile);
     if (r.error) { clearTimeout(safetyTimer); sse.sendError({message:r.error,statusCode:400}); return; }
     imageApiKey = r.key;
   }
   const result = await provider.edit({
     prompt: editPrompt,
     currentImage: { mimeType: currentMime, data: currentBase64 },
     apiKey: imageApiKey,
     model: imageModel,                                   // from style catalog
     logoImageData: editLogoData,
     additionalRefs: slide1Ref ? [slide1Ref] : undefined, // CRSL-EDIT-05 style anchor
   });
   ```

10. **Optimize + upload**:
    - `processImageWithThumbnail(result.buffer)` → `{ image, thumbnail }`.
    - Compute next version number:
      ```typescript
      const { data: existingVersions } = await supabase
        .from("post_slide_versions")
        .select("version_number")
        .eq("post_slide_id", slide_id)
        .order("version_number", { ascending: false })
        .limit(1);
      const nextVersionNumber = (existingVersions?.[0]?.version_number || 0) + 1;
      ```
    - Storage path: `${user.id}/carousel/${post_id}/slide-${slide.slide_number}-v${nextVersionNumber}-${versionId}.webp` (per RESEARCH.md "Storage path convention for slide versions").
    - Upload via `createAdminSupabase()` storage client (RESEARCH.md Pitfall 4).
    - Upload thumbnail to `${user.id}/thumbnails/carousel/${post_id}/slide-${slide.slide_number}-v${nextVersionNumber}-${versionId}.webp` (non-critical; warn-only on failure).

11. **Insert post_slide_versions row**:
    ```typescript
    sse.sendProgress("saving", "Saving slide version...", 90);
    const adminSb = createAdminSupabase();
    const { data: newVersion, error: insErr } = await adminSb
      .from("post_slide_versions")
      .insert({
        post_slide_id: slide_id,
        version_number: nextVersionNumber,
        image_url: publicUrl,
        thumbnail_url: thumbnailUrl,
        edit_prompt: edit_prompt,
      })
      .select()
      .single();
    if (insErr || !newVersion) throw new Error("Failed to save slide version");
    ```
    Note: Use `createAdminSupabase()` for the insert because RLS would otherwise require an extra round-trip; admin client mirrors what carousel-generation.service.ts does.

12. **Update `post_slides.image_url`** to the new version's URL — so the gallery cover thumbnail (if this is slide 1) and the viewer's slide array reflect the latest edit without a separate join in the client:
    ```typescript
    await adminSb.from("post_slides")
      .update({ image_url: publicUrl, thumbnail_url: thumbnailUrl })
      .eq("id", slide_id);
    ```
    Trade-off note: this is "latest wins" in `post_slides`. Older versions remain accessible via `post_slide_versions` for v2 navigation. This matches the research recommendation "no in-place updates that lose history" because the prior URL is preserved in the prior version row.

13. **Record usage + deduct credits**:
    - `const usageEvent = await recordUsageEvent(user.id, post_id, "edit", {}, { image_model: imageModel });`
    - If `!ownApiKey`: `await deductCredits(user.id, usageEvent.id, usageEvent.cost_usd_micros, usageEvent.charged_amount_micros);`
    - If `source === "quick_remake" && !ownApiKey`: `await incrementQuickRemakeCount(user.id);`

14. **Marketing tracking** — mirror edit.routes.ts lines 592–610 with `event_name: "edit"`, `event_key: "edit:slide:${newVersion.id}"`, payload includes `slide_id`, `slide_number`, `post_id`, `version_number`, `content_type: "carousel-slide"`.

15. **SSE complete**:
    ```typescript
    clearTimeout(safetyTimer);
    sse.sendComplete({
      slide_version_id: newVersion.id,
      version_number: newVersion.version_number,
      image_url: publicUrl,
      thumbnail_url: thumbnailUrl,
      slide_id,
      post_id,
      slide_number: slide.slide_number,
    });
    ```

16. **Caption handling**: Phase 13 v1 does NOT regenerate the carousel caption on a single-slide edit. The caption is unified for the carousel (set at generation time) and the user did not change the master commercial intent. SKIP the `ensureCaptionQuality` call (this differs from edit.routes.ts intentionally). Document the trade-off in the inline code comment: "Slide-level edits do not regenerate the carousel-wide caption — caption is master-text scoped (CRSL-09)."

17. **Error path**: Wrap the post-SSE body in `try/catch` mirroring edit.routes.ts lines 624–654; on error call `logSlideEditError` (local helper) and `sse.sendError({...})`. Pre-SSE errors return JSON via the outer try/catch (auth/validation/credits — see edit.routes.ts lines 655–664).

Anti-patterns to avoid (RESEARCH.md):
- Do NOT call `enforceExactImageText` (text rendering is not in scope for v1.1 carousel — CRSL-10).
- Do NOT pass `slideCount` to `checkCredits`.
- Do NOT use the user-scoped `supabase` client for the `post_slide_versions` INSERT — use `createAdminSupabase()` to bypass RLS (consistent with carousel-generation.service.ts).
- Do NOT trust `slide_number` from the client — fetch it server-side via the `slide_id` lookup.
- Do NOT skip the `posts.content_type === "carousel"` guard — sending a non-carousel post's image_url through this route would corrupt `post_slide_versions` with a slide that has no carousel.

References in carousel.routes.ts: place the new handler AFTER the existing `POST /api/carousel/generate` handler, BEFORE `export default router;`. Reuse the file's existing imports where possible; add `editSlideRequestSchema`, `getActiveImageProvider`, `processImageWithThumbnail`, `initSSE`, `recordUsageEvent`, `deductCredits`, `canUseQuickRemake`, `incrementQuickRemakeCount`, `getOpenAIApiKey`, `trackMarketingEvent` imports as needed.
